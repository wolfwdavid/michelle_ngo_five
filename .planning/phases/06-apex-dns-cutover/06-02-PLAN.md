    HUMAN-ONLY (GitHub Settings UI; Claude cannot click it). Requires Plan 06-01 done (DNS resolves
    to the four GitHub IPs).

    (a) REGISTER THE DOMAIN (CUTOVER Step 3):
        GitHub repo → Settings → Pages → Custom domain → enter exactly `michellengo.net` → Save.
        GitHub verifies the DNS from 06-01 and queues a Let's Encrypt certificate. You enter the
        domain ONCE — static/CNAME persists it into every future artifact, so re-deploys keep it

        (RESEARCH Pitfall 3). Do NOT treat greyed-out as failure before the wait; do NOT roll back early.

    (c) ENFORCE HTTPS once the checkbox is enabled:
        GitHub repo → Settings → Pages → tick "Enforce HTTPS". This makes http:// redirect to https://.

    BRANCH — if Enforce HTTPS is STILL greyed out well past ~1h AND DNS verifies green AND no blocking
    CAA exists (re-check `Resolve-DnsName michellengo.net -Type CAA`): the documented fix is to
    REMOVE the custom domain in Settings → Pages, Save, then RE-ADD michellengo.net and Save — this
    re-triggers cert issuance (RESEARCH Pitfall 3 / GitHub Docs). This is NOT a rollback. Only treat
    persistent failure as a rollback trigger per Task 3's rollback note.

    Do NOT introduce any proxy/Cloudflare (D-01) and do NOT touch MX/TXT (D-05).
  </action>
